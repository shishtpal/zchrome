zchrome [options] <command> [command-args]

GLOBAL OPTIONS:
  --url <ws-url>           Connect to existing Chrome (ws://...)
  --use <target-id>        Execute command on existing page
  --headless [new|old]     Enable headless mode (default: off)
  --port <port>            Debug port [default: 9222]
  --chrome <path>          Chrome binary path
  --data-dir <path>        User data directory for Chrome profile
  --timeout <ms>           Command timeout [default: 30000]
  --verbose                Print CDP messages
  --output <path>          Output file path (for screenshot/pdf)
  --full                   Capture full page screenshot (not just viewport)
  --provider <name>        Use cloud provider (kernel, notte, browserbase)
  --cleanup                Close cloud session when command exits

COMMANDS:
  open                     Launch Chrome with remote debugging
  connect                  Connect to running Chrome, get WebSocket URL
  screenshot [url]         Capture PNG screenshot (use -s for element)
  pdf [url]                Generate PDF
  evaluate [url] <expr>    Evaluate JS expression
  snapshot                 Capture accessibility tree of active page, save to zsnap.json
  diff <subcommand>        Compare snapshots or screenshots (snapshot/screenshot/url)
  version                  Print browser version info
  help                     Show this help message

REPL COMMANDS (interactive mode):
  interactive              REPL: enter CDP commands as JSON

NAVIGATION:
  navigate <url>           Navigate to URL, print final URL + title
  back                     Go back in history
  forward                  Go forward in history
  reload                   Reload current page

TAB MANAGEMENT:
  tab                      List open tabs (numbered)
  tab new [url]            Open new tab (optionally navigate to URL)
  tab <n>                  Switch to tab n
  tab close [n]            Close tab n (default: current)
  window new               Open new browser window
  list-targets             List all open targets
  pages                    List all open pages with target IDs

ELEMENT ACTIONS:
  click <sel>              Click element (CSS selector or @ref)
  dblclick <sel>           Double-click element
  hover <sel>              Hover over element
  focus <sel>              Focus element
  type <sel> <text>        Type text into element (appends)
  fill <sel> <text>        Clear and fill element with text
  select <sel> <value>     Select dropdown option by value
  check <sel>              Check checkbox
  uncheck <sel>            Uncheck checkbox
  scroll <dir> [px]        Scroll page (up/down/left/right) [default: 300px]
  scrollintoview <sel>     Scroll element into view (alias: scrollinto)
  drag <src> <tgt>         Drag and drop from source to target element
  upload <sel> <files>     Upload files to file input

KEYBOARD:
  press <key>              Press key (Enter, Tab, Control+a) (alias: key)
  keydown <key>            Hold key down
  keyup <key>              Release key

MOUSE:
  mouse move <x> <y>       Move mouse to coordinates
  mouse down [button]      Press mouse button (left/right/middle)
  mouse up [button]        Release mouse button
  mouse wheel <dy> [dx]    Scroll mouse wheel

CURSOR:
  cursor active            Show the currently focused element
  cursor hover             Show the element under the mouse cursor
  cursor record <file>     Record mouse/keyboard events to JSON file
  cursor replay <file>     Replay events from JSON file (with assert support)
    --retries <n>          Retries on assertion failure (default: 3)
    --retry-delay <ms>     Wait before retry (default: 100)
    --fallback <file>      Fallback JSON on permanent failure
    --resume               Resume from last successful action
    --from <n>             Start from command index n

GETTERS:
  get text <sel>           Get text content
  get html <sel>           Get innerHTML
  get dom <sel>            Get outerHTML
  get value <sel>          Get input value
  get attr <sel> <attr>    Get attribute value
  get title                Get page title
  get url                  Get current URL
  get count <sel>          Count matching elements
  get box <sel>            Get bounding box (x, y, width, height)
  get styles <sel>         Get computed styles (JSON)

WAIT:
  wait <selector>          Wait for element to be visible
  wait <ms>                Wait for time (milliseconds)
  wait --text "string"     Wait for text to appear on page
  wait --match "pattern"   Wait for URL to match glob pattern
  wait --load <state>      Wait for load state (load, domcontentloaded, networkidle)
  wait --fn "expression"   Wait for JS expression to return truthy

COOKIE MANAGEMENT:
  cookies                  List all cookies
  cookies set <n> <v>      Set a cookie
  cookies clear            Clear all cookies
  storage local            Get all localStorage entries (JSON)
  storage local <key>      Get specific localStorage key
  storage local set <k> <v> Set localStorage value
  storage local clear      Clear all localStorage
  storage local export <f> Export localStorage to JSON/YAML file
  storage local import <f> Import localStorage from JSON/YAML file
  storage session          Same commands for sessionStorage

NETWORK:
  network route <url>                Intercept requests matching URL pattern
  network route <url> --abort        Block matching requests
  network route <url> --body <json>  Mock response with JSON body
  network route <url> --file <path>  Mock response with file contents
  network route <url> --redirect <target>  Redirect to another host
  network unroute                    Remove all routes
  network requests                   View tracked requests
  network requests --clear           Clear request log
  network requests --filter <pat>    Filter by URL substring

DEVELOPER TOOLS (dev):
  dev trace start [path]             Start recording trace
  dev trace stop [path]              Stop and save trace to file
  dev trace categories               List available trace categories
  dev profiler <secs> [path]         Profile for N seconds (0=until Enter)
  dev profiler start                 Start profiling (REPL mode only)
  dev profiler stop [path]           Stop and save profile (.cpuprofile)
  dev console                        View console messages (log, error, warn, info)
  dev console --clear                Clear console messages
  dev errors                         View page errors (uncaught exceptions)
  dev errors --clear                 Clear error history
  dev highlight <selector>           Highlight element on page (3 second overlay)
  dev state save <path>              Save auth state (cookies + storage) to file
  dev state load <path>              Load auth state from file
  dev state list                     List saved state files
  dev state show <file>              Show state file summary
  dev state rename <old> <new>       Rename state file
  dev state clear [name]             Clear specific state file
  dev state clear --all              Clear all saved state files
  dev state clean --older-than <days>  Delete states older than N days

DIFF (comparing pages):
  diff snapshot                        Compare current vs last session snapshot
  diff snapshot --baseline <file>      Compare current vs saved snapshot file
  diff snapshot -s "#main" -c          Scoped snapshot diff, compact mode
  diff screenshot --baseline <file>    Visual pixel diff against baseline image
  diff screenshot -b b.png -o d.png    Save diff image to custom path
  diff screenshot -b b.png -t 0.2      Adjust color threshold (0-1)
  diff url <url1> <url2>               Compare two URLs (snapshot diff)
  diff url <url1> <url2> --screenshot  Also do visual diff between URLs
  diff url <url1> <url2> --wait-until networkidle  Custom wait strategy

SNAPSHOT OPTIONS:
  -i, --interactive-only   Only include interactive elements
  -c, --compact            Compact output (skip empty structural elements)
  -d, --depth <n>          Limit tree depth
  -s, --selector <sel>     Scope snapshot to CSS selector
  -m, --mark               Inject unique IDs (zc-1, zc-2, ...) into interactive elements

CLOUD BROWSER PROVIDERS:
  zchrome provider list    List available providers and their status
  zchrome provider set <n> Set default provider for this session
  zchrome provider status  Show current provider and session info
  zchrome provider close   Close active cloud session

  Once a provider is set, all subsequent commands use the cloud browser.
  Sessions persist between commands for faster execution.

CHROME EXTENSIONS:
  zchrome extensions               List loaded extensions
  zchrome extensions list          List loaded extensions
  zchrome extensions load <path>   Load an unpacked extension
  zchrome extensions unload <path> Unload an extension

  Notes:
  --via=port             (Default) Use --load-extension CLI flag
                         Compatible with Chrome 137+ via workaround
  --via=pipe             Use CDP Extensions.loadUnpacked via pipe
                         Future-proof method for Chrome 137+
                         (Windows, Linux, macOS)

  Notes:
  - Extensions require headed mode (headless is auto-disabled)
  - Extensions cannot be used with cloud providers
  - Restart Chrome with 'zchrome open' after loading/unloading extensions
  - Extensions are stored per-session
  - The --via setting is saved to config for subsequent commands

ENVIRONMENT VARIABLES:
  ZCHROME_PROVIDER             Default provider (local, kernel, notte, browserbase)
  ZCHROME_KERNEL_API_KEY       Kernel.sh API key
  ZCHROME_NOTTE_API_KEY        Notte.cc API key
  ZCHROME_BROWSERBASE_API_KEY  Browserbase API key
  ZCHROME_BROWSER              Chrome binary path
  ZCHROME_PORT                 Debug port
  ZCHROME_DATA_DIR             User data directory
  ZCHROME_VERBOSE              Enable verbose mode (1 or true)
  ZCHROME_HEADLESS             Headless mode (new or old)
  ZCHROME_SESSION              Default session name
  ZCHROME_EXTENSIONS           Comma-separated extension paths (merges with config)

CONFIG FILE:
  zchrome.json is stored alongside the executable for portability.
  It stores chrome_path, data_dir, port, ws_url, last_target, extensions, via, and mouse position.
  Options from command line override config file values.

  To manually configure extensions in config:
  {
    "extensions": ["/path/to/extension1", "/path/to/extension2"],
    "via": "port"
  }

EXAMPLES:
  # Launch Chrome and connect
  zchrome open --chrome "C:\Program Files\Google\Chrome\Application\chrome.exe"
  zchrome connect

  # Subsequent commands use saved WebSocket URL
  zchrome pages
  zchrome --use <target-id> screenshot --output page.png
  zchrome evaluate "document.title"

  # Launch headless and take screenshot
  zchrome open --headless
  zchrome connect
  zchrome navigate https://example.com
  zchrome screenshot --output page.png

  # Element screenshot (CSS selector or @ref)
  zchrome screenshot -s "#login-form" --output form.png
  zchrome screenshot -s @e5 --output element.png

  # Take snapshot of active page's accessibility tree
  zchrome snapshot                       # Snapshot current page
  zchrome snapshot -i                    # Interactive elements only
  zchrome snapshot -c -d 3               # Compact mode, depth 3
  zchrome snapshot -s "#main-content"    # Scope to selector
  zchrome snapshot --mark                # Inject IDs, then use #zc-<N>

  # Interact with elements (use CSS selectors or @refs from snapshot)
  zchrome click "#login-btn"             # Click by CSS selector
  zchrome click @e3                      # Click by snapshot ref
  zchrome fill "#email" "test@example.com"
  zchrome type @e5 "password"
  zchrome select "#country" "US"
  zchrome scroll down 500
  zchrome scrollinto "#footer"
  zchrome drag @e3 @e7                    # Drag element to another
  zchrome upload "#file-input" /path/to/file.pdf  # Upload file
  zchrome upload @e5 doc1.txt doc2.txt   # Upload multiple files

  # Get element information
  zchrome get text @e3                   # Get text content by ref
  zchrome get value "#email"             # Get input value
  zchrome get attr @e5 href              # Get href attribute
  zchrome get title                      # Get page title
  zchrome get url                        # Get current URL
  zchrome get count "li.item"            # Count elements

  # Keyboard input
  zchrome press Enter                   # Press Enter key
  zchrome press Control+a               # Select all
  zchrome key Tab                       # Press Tab (alias for press)
  zchrome keydown Shift                 # Hold Shift down
  zchrome keyup Shift                   # Release Shift

  # Mouse control
  zchrome mouse move 100 200            # Move mouse to coordinates
  zchrome mouse down left               # Press left button
  zchrome mouse up                      # Release button
  zchrome mouse wheel -100              # Scroll down 100px

  # Cursor info
  zchrome cursor active                 # Show focused element
  zchrome cursor hover                  # Show element at mouse position

  # Macro recording and playback
  zchrome cursor record macro.json      # Record events (Enter to stop)
  zchrome cursor replay macro.json      # Replay recorded events
  zchrome cursor replay form.json --retries 5  # With custom retries
  zchrome cursor replay form.json --fallback error.json  # With fallback

  # Wait for conditions
  zchrome wait "#login-form"            # Wait for element
  zchrome wait 2000                     # Wait 2 seconds
  zchrome wait --text "Welcome"         # Wait for text
  zchrome wait --match "**/dashboard"   # Wait for URL pattern
  zchrome wait --load networkidle       # Wait for network idle
  zchrome wait --fn "window.ready"      # Wait for JS condition

  # Developer tools
  zchrome dev console                   # View console messages
  zchrome dev errors                    # View page errors
  zchrome dev highlight "#login-btn"    # Highlight element
  zchrome dev state save login.json     # Save auth state
  zchrome dev state load login.json     # Restore auth state
  zchrome dev profiler 10 profile.cpuprofile  # Profile for 10 seconds
  zchrome dev profiler 0 profile.cpuprofile   # Profile until Enter pressed

  # Diff commands - compare pages
  zchrome diff snapshot                          # Compare vs last snapshot
  zchrome diff snapshot --baseline before.txt    # Compare vs saved file
  zchrome diff snapshot -s "#main" -c            # Scoped, compact diff
  zchrome diff screenshot --baseline before.png  # Visual pixel diff
  zchrome diff screenshot -b before.png -t 0.05  # Stricter threshold
  zchrome diff url https://v1.com https://v2.com # Compare two URLs
  zchrome diff url https://v1.com https://v2.com --screenshot  # With visual diff

  # Cloud browser providers
  # Set your API key first (PowerShell)
  # $env:ZCHROME_KERNEL_API_KEY = "your-api-key"

  zchrome provider list                          # List available providers
  zchrome provider set kernel                    # Set Kernel.sh as default

  # Create a cloud browser session (like 'open' for local Chrome)
  zchrome open                                   # Creates cloud session
  # Output: Session ID, WebSocket URL, Live view URL

  # Now run commands - they use the cloud browser
  zchrome navigate https://example.com           # Uses cloud session
  zchrome screenshot --output page.png           # Same session
  zchrome snapshot                               # Same session

  # Session persists between commands (reuses same browser)
  zchrome click "#login-btn"
  zchrome fill "#email" "test@example.com"

  # Verify session is still alive
  zchrome connect                                # Shows session info

  # View provider status
  zchrome provider status

  # Close session when done
  zchrome provider close

  # Or use --cleanup to close after single command
  zchrome --cleanup navigate https://example.com

  # Switch back to local Chrome
  zchrome provider set local
  zchrome open                                   # Now launches local Chrome

  # Chrome extensions
  zchrome extensions load /path/to/my-extension # Load an unpacked extension
  zchrome extensions list                        # List loaded extensions
  zchrome open                                   # Restart Chrome with extension
  zchrome extensions unload /path/to/my-extension # Remove extension

  # Load extensions via environment variable
  # $env:ZCHROME_EXTENSIONS = "/ext1,/ext2"
  zchrome open                                   # Opens with extensions loaded
